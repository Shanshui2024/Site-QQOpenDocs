    # 错误码介绍

    错误码分为两部分：

    • http status code 2XX / 4XX / 5XX
    • http responese body: {code: "", message: ""} 具体业务返回码

    说明

    当 http code 不是 2XX 的时候，http 的 responese body 中还会有平台的错误码信息。

    如：

    # HTTP状态码

    值 含义
    200 成功
    204 成功，但是无包体，一般用于删除操作
    201,202 异步操作成功，虽然说成功，但是会返回一个 error body，需要特殊处理
    401 认证失败
    404 未找到 API
    405 http method 不允许
    429 频率限制
    500 处理失败
    504 处理失败
